# Governance

## Maintainer Model

CC Desk uses a founder-maintainer model. The repository owner retains final responsibility for product direction, project name, releases, security disclosures, and maintainer appointments. Maintainers handle day-to-day triage, reviews, documentation, releases, and dependency updates.

## Decisions

- Small fixes, documentation, tests, and routine dependency updates may be merged after required CI passes.
- Changes affecting user data, compatibility, permissions, security boundaries, release infrastructure, or product scope require a written design discussion before implementation.
- Substantial decisions are recorded in a GitHub Issue, Discussion, pull request, or public documentation so contributors can understand the rationale.
- During an active security incident, maintainers may make a minimal private fix first and document the decision after coordinated disclosure.

## Becoming or Leaving a Maintainer

Potential maintainers are nominated based on sustained, constructive contributions and demonstrated review judgment. The repository owner appoints or removes maintainers when needed to protect project health, security, or continuity. Maintainers may step down at any time by notifying the repository owner through an agreed private channel. The temporary `[conduct]` reporting path in the [Code of Conduct](CODE_OF_CONDUCT.md) is not a maintainer-administration channel.
